\# Markdown Notes

> Markdown 是一种轻量级标记语言，可以使用简单的符号编写格式清晰的文档。  

\---



\## 1. 标题



Markdown 支持六级标题，在文字前添加 `#` 即可。



```markdown

\# 一级标题



\## 二级标题



\### 三级标题



\#### 四级标题



\##### 五级标题



\###### 六级标题

```



建议在 `#` 和标题文字之间保留一个空格。



\---



\## 2. 普通段落



直接输入文字即可形成段落。



```markdown

这是第一段文字。



这是第二段文字。

```



段落之间需要保留一个空行。



\---



\## 3. 换行



\### 3.1 使用两个空格换行



在一行末尾输入两个空格，然后按回车。



```markdown

第一行末尾有两个空格。  

这是第二行。

```



\### 3.2 使用 HTML 标签换行



```markdown

第一行。<br>

第二行。

```

\## 4. 粗体、斜体和删除线



\### 4.1 粗体



```markdown

\*\*这是粗体文字\*\*



\_\_这也是粗体文字\_\_

```



显示效果：



\*\*这是粗体文字\*\*  





\### 4.2 斜体



```markdown

\*这是斜体文字\*



\_这也是斜体文字\_

```



显示效果：



\*这是斜体文字\*



\### 4.3 粗斜体



```markdown

\*\*\*这是粗斜体文字\*\*\*



\_\_\_这也是粗斜体文字\_\_\_

```



显示效果：



\*\*\*这是粗斜体文字\*\*\*



\### 4.4 删除线



```markdown

\~\~这是被删除的文字\~\~

```



显示效果：



\~\~这是被删除的文字\~\~



> 删除线属于扩展语法，大多数 Markdown 编辑器和平台都支持。

\## 5. 无序列表



使用 `-`、`+` 或 `\*` 创建无序列表。



```markdown

\- 苹果

\- 香蕉

\- 橙子

```



也可以这样写：



```markdown

\* 苹果

\* 香蕉

\* 橙子

```



或：



```markdown

\+ 苹果

\+ 香蕉

\+ 橙子

```



* 苹果  
* 香蕉  
* 橙子



建议统一使用 `-`，方便保持文档风格一致。

\## 6. 多级列表



通过缩进创建嵌套列表，通常使用两个或四个空格。



```markdown

\- 前端开发

&#x20; - HTML

&#x20; - CSS

&#x20; - JavaScript

\- 后端开发

&#x20; - Java

&#x20; - Python

&#x20; - Go

```



有序列表也可以嵌套：



```markdown

1\. 安装开发工具

&#x20;  1. 安装编辑器

&#x20;  2. 安装运行环境

2\. 创建项目

&#x20;  1. 创建目录

&#x20;  2. 创建文件

```



无序列表和有序列表也可以混合使用：



```markdown

1\. 学习 HTML

&#x20;  - 标签

&#x20;  - 表单

&#x20;  - 语义化

2\. 学习 CSS

&#x20;  - 选择器

&#x20;  - 盒模型

&#x20;  - 布局

```

\## 8. 任务列表



任务列表常用于待办事项。



```markdown

\- \[x] 学习标题语法

\- \[x] 学习列表语法

\- \[ ] 学习表格语法

\- \[ ] 完成练习

```



其中：



\- `\[x]` 表示已经完成

\- `\[ ]` 表示尚未完成



> 任务列表属于扩展语法，GitHub、GitLab、Typora 等平台通常都支持。

\## 9. 引用



使用 `>` 创建引用。



```markdown

> 这是一段引用文字。

```



\### 9.1 多行引用



```markdown

> 这是引用的第一行。

>

> 这是引用的第二段。

```



\### 9.2 嵌套引用



```markdown

> 一级引用

>

>> 二级引用

>>

>>> 三级引用

```



\### 9.3 引用中使用其他语法



```markdown

> ## 引用中的标题

>

> - 列表项目一

> - 列表项目二

>

> \*\*引用中的粗体文字\*\*

```

\## 10. 行内代码



使用一对反引号包裹代码。



```markdown

使用 `print()` 函数输出内容。

```



显示效果：



使用 `print()` 函数输出内容。



如果内容本身包含反引号，可以使用两个反引号包裹：



```markdown

``代码中包含 ` 反引号``

```

\## 11. 代码块



使用三个反引号包裹多行代码。



````markdown

```

这里是代码内容

这里是第二行代码

```

`````



\### 11.1 指定编程语言



在开头的三个反引号后添加语言名称，可以启用语法高亮。



````markdown

```python

def greet(name):

&#x20;   print(f"Hello, {name}!")



greet("Markdown")

```

````  





JavaScript 示例：



````markdown

```javascript

function add(a, b) {

&#x20; return a + b;

}



console.log(add(10, 20));

```
````

\## 13. 分隔线



使用三个或更多的 `-`、`\*` 或 `\_` 创建分隔线。



```markdown

\---



\*\*\*



\_\_\_

```



为了避免与其他语法混淆，建议在分隔线前后各保留一个空行。

\## 14. 超链接



\### 14.1 普通链接



```markdown

\[OpenAI 官网](https://openai.com)

```



基本格式：



```text

\[链接文字](链接地址)

```



\### 14.2 带提示文字的链接



```markdown

\[OpenAI 官网](https://openai.com "访问 OpenAI")

```



鼠标悬停在链接上时，可能会显示提示文字。





\---



\## 15. 引用式链接



当同一个链接需要多次使用时，可以采用引用式写法。



```markdown

这是一个指向 \[OpenAI]\[openai] 的链接。



这里再次访问 \[OpenAI]\[openai]。



\[openai]: https://openai.com "OpenAI 官网"

```



引用标识通常不区分大小写。



也可以省略引用标识：



```markdown

访问 \[OpenAI]\[]。



\[OpenAI]: https://openai.com

```

\## 16. 图片



图片语法与链接类似，只是在前面增加一个感叹号 `!`。



```markdown

!\[图片说明](https://example.com/image.png)

```



基本格式：



```text

!\[替代文字](图片地址)

```



\### 16.1 带提示文字的图片



```markdown

!\[图片说明](https://example.com/image.png "图片标题")

```



\### 16.2 使用本地图片



假设 Markdown 文件和 `images` 文件夹位于同一个目录：



```markdown

!\[项目截图](./images/screenshot.png)

```



上一级目录中的图片：



```markdown

!\[项目截图](../images/screenshot.png)

```



\### 16.3 点击图片跳转



可以将图片语法放进链接语法中：



```markdown

\[!\[图片说明](https://example.com/image.png)](https://example.com)

```

\## 17. 表格



使用竖线 `|` 和短横线 `-` 创建表格。



```markdown

| 姓名 | 年龄 | 城市 |

| --- | --- | --- |

| 张三 | 20 | 北京 |

| 李四 | 22 | 上海 |

| 王五 | 21 | 广州 |

```



\### 17.1 表格对齐



冒号 `:` 用于控制对齐方式。



```markdown

| 左对齐 | 居中对齐 | 右对齐 |

| :--- | :---: | ---: |

| 内容 A | 内容 B | 内容 C |

| 内容 D | 内容 E | 内容 F |

```



对应规则：



```text

:---   左对齐

:---:  居中对齐

\---:   右对齐

```



\### 17.2 简化表格



左右两侧的竖线通常可以省略：



```markdown

姓名 | 年龄 | 城市

\--- | --- | ---

张三 | 20 | 北京

李四 | 22 | 上海

```



为了提高源码可读性，建议保留两侧竖线。



\### 17.3 在表格中输入竖线



使用反斜杠转义：



```markdown

| 表达式 | 含义 |

| --- | --- |

| `A \\| B` | A 或 B |

```

\## 18. 转义字符



当需要显示 Markdown 特殊符号本身时，可以在符号前添加反斜杠 `\\`。



```markdown

\\\*这段文字不会变成斜体\\\*



\\# 这行文字不会变成标题



\\- 这行文字不会变成列表



\\\[这不是链接\\](https://example.com)

```



常见可转义字符：



```text

\\  反斜杠

`  反引号

\*  星号

\_  下划线

{} 花括号

\[] 方括号

() 圆括号

\#  井号

\+  加号

\-  减号

.  英文句点

!  感叹号

|  竖线

>  大于号

```

\## 19. 脚注



脚注可以用来补充解释或标注资料来源。



```markdown

Markdown 是一种轻量级标记语言。\[^1]



这里还可以添加另一个脚注。\[^note]



\[^1]: Markdown 由 John Gruber 创建。

\[^note]: 脚注内容通常会显示在文档底部。

```



较长的脚注可以使用缩进：



```markdown

这是一个较长的脚注。\[^long]



\[^long]: 这是脚注的第一段。



&#x20;   这是脚注的第二段，需要进行缩进。

```



> 脚注属于扩展语法，不同平台的支持情况可能不同。

\## 21. HTML 标签



多数 Markdown 解析器允许直接使用部分 HTML 标签。



\### 21.1 折叠内容



```html

<details>

<summary>点击查看详细内容</summary>



这里是折叠起来的内容。



\- 项目一

\- 项目二



</details>

```



\### 21.2 下划线



Markdown 本身没有统一的下划线语法，可以使用 HTML：



```html

<u>这是带下划线的文字</u>

```



\### 21.3 上标和下标



```html

X<sup>2</sup>



H<sub>2</sub>O

```



\### 21.4 键盘按键



```html

按下 <kbd>Ctrl</kbd> + <kbd>C</kbd> 复制内容。

```



按下 <kbd>Ctrl</kbd> + <kbd>C</kbd> 复制内容。





\### 21.5 文字居中



```html

<p align="center">这段文字居中显示</p>

```



> 部分平台会限制或过滤 HTML 标签。

\## 22. 注释



Markdown 没有统一的注释语法，通常可以使用 HTML 注释。



```html

<!-- 这是一段注释，不会显示在最终页面中。 -->

```



多行注释：



```html

<!--

这里是多行注释。

这些内容通常不会被渲染出来。

\-->

```

\## 26. 高亮文字



部分编辑器支持使用双等号高亮文字。



```markdown

这是普通文字，==这是高亮文字==。

```



> 高亮不是标准 Markdown 语法，Obsidian、Typora 等工具可能支持。

\## 29. 相对路径



常见相对路径写法：



```text

./file.md          当前目录中的文件

../file.md         上一级目录中的文件

../../file.md      上两级目录中的文件

./images/logo.png  当前目录下 images 文件夹中的图片

/docs/guide.md     从站点或项目根目录开始的路径

```



在 GitHub 仓库、静态网站和本地编辑器中，根路径的处理方式可能不同。
